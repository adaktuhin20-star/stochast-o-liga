# 🌌 STOCHAST-O-LIGA | Global Extraction Network

A real-time, synchronized web application built to host interactive events for a departmental statistics fest. The platform features a unified login portal that seamlessly connects teams, audiences, a main stage projector, and a master admin console via low-latency Firebase WebSockets.

## 🚀 Event Modules & Official Rules

The application currently supports two major interactive rounds, each with its own dedicated logic, UI, and live-sync engine.

### Round 2: STAT CIPHER (Pathfinding & Survival)
*"Discover the elusive treasure behind the correct door."*

**Event Rules:**
1. In this round each team will be provided with **6 clues** where the preceding clue will lead to the door of the next one.
2. Several doors hide the same clue, so a team can make a **random lucky guess**! No points will be awarded for a lucky guess, but it can lead you to the treasure in much fewer steps. *(e.g., If a lucky guess on clue 2 leads to clue 6, you bypass the middle nodes).*
3. Opening the wrong door results in **loss of life**. Each team has exactly **3 lives** for the entire journey.
4. For each correct door you open, **5 points** will be awarded. 
5. Upon unlocking the final treasure door, **10 extra points** will be awarded.
6. The absolute time limit for the entire journey is **15 minutes**.

---
### Round 3: BLITZ (Rapid-Fire Buzzer)
*"Feel the heat as you blaze through rapid-fire rounds of Statistical trivia."*

**Event Rules:**
1. We will be asking **15 Rapid Fire questions** which you will be prompted to answer if you hit the buzzer. Teams will use the devices of their respective team coordinators.
2. Only the **first team to hit the buzzer** will be allowed to answer. Please make sure you do not answer until prompted to do so.
3. If a team presses the buzzer **before the question is completed**, the text dictation stops instantly across all screens. The remaining part of the question won't be dictated, and the team must answer based on the partial question.
4. Every correct answer will be rewarded with **5 points**. 
5. Every incorrect answer (or in case the question is unanswered even after hitting the buzzer) will lead to a **deduction of 5 points**.

**System Mechanics:** * **Live Dictation:** The question types out word-by-word synchronously on the Main Stage Projector and Team Terminals at a natural speaking speed.
* **Instant Lockout:** Firebase transactions guarantee that even if multiple teams tap within the same millisecond, only the first signal is registered, instantly locking out all other buzzers and triggering the team's specific audio cue.

## 🔑 System Access & Routing Passcodes

The entire ecosystem runs through a single master login portal (`index.html`). The system uses hardcoded passcodes to identify the user's role and route them to the correct dashboard for the selected round.

* **Admin / Quizmaster Console:** Enter passcode `masterkey`
* **Main Stage Projector View:** Enter passcode `adminstage`
* **Public / Audience NOC View:** Enter passcode `publicview`
* **Participating Teams:** Enter the dynamic, custom passcodes generated by the Admin during team registration.

---

## 🏗️ System Architecture & Views

* **`index.html` (Network Access):** The master login portal. Users select the protocol (Round 2 or Round 3) from a dropdown and enter their designated passcode.
* **Player Terminals (`player.html` / `blitz_player.html`):** The dashboard for participating teams. Includes interactive grid paths, countdown timers, live scores, and ultra-fast WebSocket buzzer mechanisms.
* **Master Controls (`admin.html` / `blitz_admin.html`):** The Quizmaster's dashboard. Features team registration, a 20-question bank configurator, dictation/playback controls, network pause/play toggles, and live scoreboards.
* **Main Stage (`projector.html` / `blitz_projector.html`):** A massive, cinematic UI designed for the live audience projector. Displays real-time racing progress bars, synchronous dictation text, and dramatic winner lockout screens.
* **NOC Observer (`audience.html`):** A specialized view for the audience (Stat Cipher only) to track live event logs, network anomalies, and a real-time survival leaderboard.

## 🛠️ Tech Stack
* **Frontend:** HTML5, vanilla JavaScript (ES Modules)
* **Styling:** Tailwind CSS (via CDN) for rapid, responsive UI development.
* **Icons:** Lucide Icons.
* **Math Rendering:** KaTeX (for flawlessly rendering statistical formulas, equations, and LaTeX dynamically during live dictation).
* **Backend / Database:** Firebase Realtime Database (v10 Modular SDK) for sub-second state synchronization.
* **Audio:** Native HTML5 Audio API for custom buzzer and interaction sound effects.

## ⚙️ Installation & Setup

### 1. Clone the Repository
```bash
git clone https://github.com/yourusername/stochast-o-liga.git
cd stochast-o-liga
```

### 2. Configure Firebase Environment
This application relies entirely on Firebase Realtime Database for state management.
1. Create a new project in the [Firebase Console](https://console.firebase.google.com/).
2. Enable **Realtime Database** and set the security rules to `true` for initial testing (ensure you write secure rules for production).
3. Open the `.html` files in your repository and locate the `<script type="module">` tags near the bottom.
4. Replace the placeholder `firebaseConfig` object with your actual Firebase project credentials:
```javascript
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
    projectId: "YOUR_PROJECT_ID",
    storageBucket: "YOUR_PROJECT_ID.firebasestorage.app",
    messagingSenderId: "YOUR_SENDER_ID",
    appId: "YOUR_APP_ID"
};
```

### 3. Media Assets Required
Ensure the following files are present in the root directory alongside your HTML files to prevent broken UI or missing sounds:
* `logo.png`
* `inferencia_logo.png`
* `correct answer.mp3`
* `wrong answer.mp3`
* `eritnhut1992-buzzer-or-wrong-answer-20582.mp3`

### 4. Running the Application
Because the application imports ES Modules natively (`type="module"`), it must be served over HTTP/HTTPS (it will fail via `file://` protocol due to CORS).
* **Local Development:** Use an extension like VS Code's "Live Server" or run a simple Python server:
  ```bash
  python -m http.server 8000
  ```
* **Production:** Deploy the directory directly to Vercel, Netlify, Firebase Hosting, or GitHub Pages.
